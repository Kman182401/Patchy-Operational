# Execution Playbook

## Before Editing
1. Re-read the approved plan.
2. Re-read relevant repo instructions and local conventions.
3. Inspect the exact files, tests, configs, and current git state that the plan touches.
4. Confirm whether any external facts must be re-verified.

## During Execution
1. Work milestone by milestone.
2. Prefer solving the root cause, not only the visible symptom.
3. Preserve unrelated user changes.
4. Keep changes easy to review.

## When New Information Changes The Plan
1. Stop the affected implementation path.
2. Record the mismatch between the plan and reality.
3. Update the plan explicitly before resuming.
4. Do not silently drift away from the agreed contract.

## Failure Handling
1. Capture the exact error or failing behavior.
2. Inspect local evidence first.
3. Research current official guidance if the failure touches external tools or versions.
4. Change approach only after you can explain why the new path is better grounded.
